# ADR 0007 - Shared Cache Snapshot Governance

**Category:** GatingRule
**Date:** 2026-01-04
**Status:** Proposed

## Context
- Warm runs benefit from shared cache snapshots
- Snapshots can become stale or incompatible

## Decision
- Require snapshot validation before import
- Auto-fallback to tracked seed when missing

## Consequences
- Warm runs always start with valid cache state
- Extra validation overhead at startup
